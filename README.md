## 💬 提示词对话日志 & 制作过程

### 第一轮对话

> **用户：** 你是数据分析专家，我想做一个关于咖啡店每天不同时间段营业额收入的热力图 请你模拟3月份的营业额数据 需要具体到时间戳（客人是什么时候下单的）

**我的做法：**
- 创建了一个完整的 HTML 页面，使用 ECharts 绘制热力图
- 模拟了12种饮品，每小时客流权重，周几权重
- 生成了订单数据（含精确时间戳），并渲染热力图和柱状图
- 营业时间设为 7:00-21:00

**问题：** 数据模拟较为粗糙，时段客流分布不够真实，周末反而比工作日客流更多，与实际不符。

---

### 第二轮对话

> **用户：** 你是经营咖啡店多年的专家，请你参考市面上大部分咖啡店的具体到每个时间段（每个小时）的营业情况，去模拟这个数据，更接近现实更好，然后再去做这个热力图

**我的做法：**
- 搜索了真实的咖啡店运营数据（Kaggle Coffee Shop Sales、Maven Coffee 等数据集）
- 从多个真实分析报告中提取关键规律：
  - 早高峰 7:00-10:00 贡献全天约38%营业额
  - 工作日营业额占74%，周末占26%
  - 8:00-9:00 为绝对峰值
- 重新设计了时段客流模型：
  - 工作日和周末分别设置不同的每小时订单数
  - 加入周几微调系数（周五最高，周二最低工作日）
  - 使用泊松分布模拟随机波动
  - 加入天气因素（25%阴雨+10%，5%恶劣-15%）
  - 早高峰有概率加购早餐（可颂、三明治等）

**问题：** 虽然数据模型更真实了，但热力图视觉上高峰和低谷的差异不够明显，用户反馈「没体现出这些高峰的数据」。

---

### 第三轮对话

> **用户：** 更真实的时段客流模型：早高峰8:00-10:00为绝对峰值（占全天约38%），9:00为最高峰...那你这个热力图怎么没体现出这些高峰的数据呢

**原因分析：**
1. **客流量级偏保守** — 工作日全天才180单，8-9点才25单/小时，对一家正经咖啡店来说太少了
2. **热力图视觉问题** — 14列×31行格子太密，色阶过渡不够鲜明，高峰红和低谷蓝挤在一起难以区分

**我的改进：**
1. **大幅提升客流量到真实水平**：
   2. 工作日日均约300单（之前180单）
   3. 9:00 绝对峰值 55单/小时（之前25单）
   4. 8:00 第二高峰 45单/小时（之前22单）
   5. 15:00 下午茶高峰 24单/小时
2. **热力图视觉优化**：
   7. 11级色阶：从纯黑(¥0) → 暗蓝 → 青绿 → 黄 → 橙 → 红 → 深红(峰值)
   8. 格子间距加大（2px深色边框），避免色块糊成一片
   9. 高亮效果：hover时出现橙色光晕
   10. Y轴标注加上星期几
3. **柱状图颜色区分**：
   12. 9:00 深红色(绝对峰值)、8:00 橙色、15:00 黄色
   13. 高于¥500的柱子顶部自动标注数值
4. **新增工作日vs周末对比图**，直观展示两类日子的时段差异

---

### 第五轮对话

> **用户：** 你是改了数据吗 怎么图跟原来不一样了

**原因分析：** HTML 中的数据每次刷新都重新用 `Math.random()` 随机生成，所以每次打开图都不一样。而且之前生成 CSV 时用的是不同的随机序列，导致 HTML 和 CSV 数据也对不上。

**我的改进：**
- 用 **Mulberry32 固定种子随机数生成器** 替换 `Math.random()`
- 种子值设为 `20260301`（代表2026年3月1日）
- HTML 和 CSV 使用完全相同的种子和算法，确保：
  - 每次打开页面数据完全一致
  - CSV 导出的数据与热力图展示的完全对应

## 🛠️ 技术栈

- **可视化**：[ECharts 5.5] — 热力图、柱状图、折线图
- **数据生成**：JavaScript（Mulberry32固定种子随机数 + 正态分布 + 权重选择）
- **数据一致性**：HTML和CSV使用同一固定种子 `20260301`，每次打开数据完全相同
- **数据导出**：Node.js → CSV（带BOM的UTF-8，Excel兼容）
- **样式**：纯CSS（暗色主题，渐变色阶）
